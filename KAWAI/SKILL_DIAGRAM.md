# Diagram — 図解生成プロンプト作成スキル

記事・スライド・SNS向けの図解（フロー図・比較表・マトリクス・リスト等）の画像生成AIプロンプトを作成する。画像そのものは作らない。

## 必読ドキュメント（実行前に必ず読む）

1. このSKILL.md内の守るべきラインと設計プロセスに従う

## 参照ファイル
- `references/diagram-types.md` — 図解型カタログ（D1〜D8）。型指定があればそれを起点にする
- `references/anti-ai-design.md` — 脱AIデザイン共通原則

## KGI

**記事の魅力とわかりやすさの最大化**。読者がセクションの要点を一目で掴み、続きを読みたくなること。

## 守るべきライン

- **テキスト: 全て日本語（タイトル・ラベル・注釈・ボトムバー含む全要素）。英語のみの図解は禁止**
- フォント: Noto Sans JP Bold以上
- 要素数: 多くても6以内。削れるなら削る
- 1要素10文字以内
- 160×90pxでも構造が伝わるか脳内プレビュー
- 装飾はプロジェクトの方針に従う
- **図解は「図」がメイン。テキストだけの画像は図解ではない。アイコン・形状・空間配置で構造を伝える**
- **初回で最良案を出す。「これがベスト？」と聞かれて変えるのは設計不足**
- **既存のロゴは生成しない**。企業ロゴ・プロダクトロゴ（Claude, OpenAI, Google等）を画像生成AIで描かせると不正確になる。ロゴが必要な場合はテキストラベルで代替するか、ロゴなしの構図で設計する

## 脱AIデザインゲート（共通原則: `references/anti-ai-design.md`）

図解プロンプト生成時に以下をチェック:

- [ ] アイコン＋ラベルの羅列で図解を済ませていないか（情報の関係性を構造で表現する）
- [ ] 全要素が同じサイズ・同じ色になっていないか（重要度で差をつける）
- [ ] 装飾過多（グラデーション背景・影・光彩）になっていないか
- [ ] ダークトーン＋ネオンカラーになっていないか
- [ ] 英語ラベルだけの図解になっていないか（全て日本語は既存ルール）

## 図解の型（毎回最適なものを選ぶ）

| 型 | 用途 |
|---|---|
| フロー | 手順・プロセス・変化の流れ |
| 比較 | Before/After・A vs B |
| ピラミッド | 階層・優先順位 |
| マトリクス | 2軸での分類・ポジショニング |
| リスト | 並列の要素・チェックリスト |
| 放射 | 中心概念と派生要素 |
| その他自由 | 上記に当てはまらない構造 |

## 出力ルール

- ファイルには**画像生成AIにそのままペーストできるプロンプトだけ**を書く
- メタ情報（目的、設計意図等）はファイルに含めない
- 複数図解がある場合は `---` で区切る
- 記事プロジェクト配下の場合: `{記事フォルダ}/図解.md` に保存

## 図解の数量ルール

- **記事の場合: 最低でもセクション（H2）ごとに1枚の図解を生成する**
- セクションの内容が図解に向かない場合（冒頭の導入、まとめ等）は省略可
- 1セクションで複数の構造がある場合は複数枚作ってよい
- サムネイルと配色・トーンを統一する

## 設計プロセス（内部実行）

1. 記事の全セクション（H2）を洗い出す
2. 各セクションの核心を1文で言語化する
3. **その核心を「図で」どう伝えるかを先に決める。テキスト配置ではなく、視覚的メタファー・アイコン・空間関係で構造を表現する方法を考える**
4. その構造に最適な図解の型を選ぶ
5. 配色・背景・レイアウトを記事のトーンと読者体験から設計する
6. 英語プロンプトを作成する（日本語テキストは""で囲む）
7. 図解ごとに対応セクション名をコメントで記す（例: `<!-- H2: AIが「見て、考えて、クリックする」時代 -->`）
8. **事実検証**: プロンプト内の数字・企業名・因果関係・構造が元ソースと一致しているか確認。不正確があれば修正

### SNS用図解の追加原則
- SNS図解の目的は**1秒で構造を伝える**こと。説明的にならない
- 参照プロンプトのフォーマットに寄せず、毎回コンテンツの本質から最適な視覚表現をゼロ設計する
- 制約やテンプレに縛られず「この内容を最も理解しやすくする図は何か」だけを考える

## Skywork生成フロー（ユーザーから指示があった場合）

プロンプト作成後、Skyworkで画像生成まで一気通貫で実行する。

1. `https://skywork.ai/?skill_id=119` にアクセス（画像生成スキル直行URL）
2. 「画像」タブが選択されていることを確認
3. プロンプト入力エリア（プレースホルダー: 「作りたい画像を説明するか…」）をクリック
4. `browser_type` の `fill` でプロンプト全文を貼り付け（`submit: false`）
5. 送信ボタン（入力エリア右下のアイコン）をクリック
6. `browser_wait_for` で生成完了を待つ（初回45秒、以降30秒刻みで確認）
7. 「タスクが完了しました」が表示されたら次のプロンプトへ進む
8. 複数プロンプトがある場合は同一プロジェクト内で連続送信する

### 注意点
- 入力エリアは contenteditable な paragraph 要素。`fill` で入力し、送信ボタンを別途クリックする
- 生成中は「生成中...」→「ツールを使用」→完了の順に遷移する
- 送信ボタンの ref は毎回変わるため、snapshot で最新の ref を取得してからクリックする
