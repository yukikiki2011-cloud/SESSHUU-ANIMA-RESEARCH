# 脱AIデザイン共通原則

AIが生成するビジュアル（Web・スライド・サムネ・図解・画像）に共通する「型」を崩す設計指針。
全ビジュアル系スキルで参照する。

---

## AIっぽさの正体

AIは「均一・対称・整頓・Web文化圏のデザイン」を出す。具体的には:

| AI臭い特徴 | なぜ起きるか |
|---|---|
| ダークトーン＋ネオンカラー | 学習データのWeb/SaaSデザイン偏重 |
| ドロップシャドウ付きカードの連打 | Material Design/Tailwindの影響 |
| 英語ラベル見出し（FEATURES, PRICING） | 英語圏スライド・LPの模倣 |
| 絵文字（🎨✅📱） | チャット文化の混入 |
| 全要素が均一サイズ・均等配置 | グリッドシステムの過剰適用 |
| アイコン＋テキストカードで図解を代替 | 構造的な図解（フロー・マトリクス）の学習不足 |
| 主張文なし（カテゴリラベルだけ） | Webのセクションヘッダー文化 |
| 均一な角丸・均一な余白 | CSSフレームワークのデフォルト値 |
| テカリ・光沢・過剰なグラデーション | AI画像生成の癖 |
| 完璧すぎる対称性 | 人間の不完全さの欠如 |

---

## 共通禁止パターン（全メディア）

1. **ダーク背景＋ネオンアクセント禁止** — ホワイトベースを基本とする
2. **ドロップシャドウ付きカードの3枚以上連打禁止** — カード以外の情報表示（テーブル・リスト・引用・図解）を混ぜる
3. **英語ラベル見出し禁止** — 日本語で主張文を書く
4. **絵文字禁止** — ビジネスコンテンツでは使わない
5. **全要素均一配置禁止** — サイズ・余白・構造にバリエーションを持たせる
6. **01/02/03ナンバリングの乱用禁止** — 手順・ステップ以外には使わない

## メディア別の追加原則

### Web / LP（12原則）

1. **セクション構造のリズムを崩す** — 全セクション同一構造の繰り返し禁止。構造を変え、CTAは2〜3箇所に絞り、英字ラベルを全セクションに付けない。セクション間の余白を不均一に
2. **グリッドの均一性を壊す** — 大1枚+小2枚の非対称グリッド。カードの中身にバリエーション。項目数を3や4の倍数に揃えない
3. **ナンバリング乱用禁止** — 01/02/03は手順にだけ。並列概念にはアイコンや太字キーワード
4. **文字に変化をつける** — 見出しのウェイトを場所で変え、1箇所だけ極端に大きい文字。行間もセクションの性格で変える
5. **色に意外性を1つ** — 背景の白に色味（暖白・冷白）。アクセントは感情を動かす場所だけ。背景色3種以上
6. **余白に呼吸** — 重要セクションは2倍以上の余白。左右非対称を1箇所。均一間隔禁止
7. **アニメーション均一禁止** — 重要要素にだけ（最大3箇所）。情報セクションはアニメなし。ホバーはクリック可能要素だけ
8. **完璧な構造を崩す** — テーブル・リスト・引用をカードと混ぜる。全幅画像や引用帯で構造を断ち切る
9. **写真の扱い** — 実写を使い、画像サイズをバラバラに。カードからはみ出す画像。AI生成をメインビジュアルに使わない
10. **マイクロコピーに人間味** — CTAを具体的に（「60分の無料ヒアリングを予約」）。敬語密度を下げ、1箇所口語を入れる
11. **角丸を統一しすぎない** — ボタン小/カード中/バッジ丸。角丸と直角を混在
12. **ヘッダー・フッターのテンプレ感を消す** — ナビ3〜4項目。フッターに人格（一言メッセージ、個人SNS）

### スライド
- 各スライドに**カテゴリラベルではなく主張文**（そのスライドで言いたいことを1行で言い切る）
- 5枚以上連続で同じレイアウトにしない
- 図解が必要な場面では**アイコン＋カードで代替せず、構造的な図解**（フロー・マトリクス・ベン図・ピラミッド等）を使う
- 背景はホワイトベース、テキストはダークグレー
- 情報密度を恐れない（Web的なスカスカより、実務資料らしい詰まった感じ）

### サムネイル / アイキャッチ
- テカリ・光沢・過剰なグラデーション禁止
- 完璧な対称レイアウトを避ける
- テキストサイズは3段階（大/中/小）に明確に分ける。中間サイズ（50-70%）を作らない
- ストック写真風の均質な画像を避ける
- 要素を詰め込みすぎない（0.3秒で認識できるか）

### 図解 / ダイアグラム
- アイコン＋ラベルの羅列で図解を済ませない
- 情報の関係性（因果・階層・比較・時系列）を**構造で表現**する
- 全要素を同じサイズ・同じ色にしない — 重要度で差をつける
- 装飾過多にしない（グラデーション背景・影・光彩は控えめに）

### 画像生成プロンプト
- 「完璧すぎる肌・完璧すぎる照明」を指示しない — 自然な質感を指定
- 均一な背景色ベタ塗りを避ける — テクスチャ・グレインを加える
- 全要素が中央揃えの構図を避ける — 三分割法・黄金比を使う
- 「プロフェッショナル」「モダン」「クリーン」だけの曖昧指示を避ける — 具体的なスタイル参照を入れる

## 一言でまとめると

> AIは「均一・対称・整頓」が得意。だから「不均一・非対称・例外」を意図的に入れる。
> AIは「Web文化圏」で育った。だから「その媒体固有の文化圏ルール」をこちらから渡す。
