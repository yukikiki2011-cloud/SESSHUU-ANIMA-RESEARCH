## 🏗️ アーキテクチャ（技術スタック）

```
project/
├── index.html          # エントリーポイント
├── slides.js           # 全スライドデータ（PRESENTATION オブジェクト）
├── app.js              # レンダリング・ページ遷移・サイドバー制御
├── style.css           # レスポンシブ対応（16:9維持）＋ アニメーション
└── ~/.claude/skills/slide-builder/
    ├── SKILL.md         # AIへの実行指示書（構成承認→生成→検証）
    ├── references/      # デザイン原則・図解カタログ（レイアウトパターン）
    └── scripts/
        └── validate-slide-deck.mjs  # 自動検査スクリプト（はみ出し・禁則チェック）
```

### データ駆動型設計
- `slides.js` がスライドの全データ（テキスト、レイアウト指示、ノート）を `PRESENTATION` オブジェクトとして保持
- `app.js` がデータのレンダリング、ページ遷移（クリック・矢印キー）、サイドバーメニュー、トークスクリプト表示を管理
- `style.css` がレスポンシブ対応のレイアウト（16:9維持）やアニメーションを定義

## 📝 スライド内容サマリー（全30枚）

### Section 1: "うまい資料"の正体（デザインの3ルール）
- **Slide 1-5**: イントロダクションとデザインの原則
- 3つのルール: **「揃える」「絞る」「余白を取る」**
- 資料作成の時間を「見た目」ではなく「中身」に使うための自動化提案

### Section 2: Claude Codeによる資料制作の実演（3ステップ）
- **Slide 6-15**: ライブデモのステップ解説
- HTMLベースのスライド → リンク・音・操作演出が自由自在
- 目的・相手・制約を入力するだけで、構成→デザイン→品質チェックまで自動実行

### Section 3: プロっぽく仕上げるテクニック（スキルの資産化）
- **Slide 16-25**: 仕上げテクニック
- 単一プロンプトではなく、以下に切り分けて資産化:
  - `SKILL.md`: AIへの実行指示書
  - `references/`: デザイン原則・改行ルール等
  - `validator`: 自動検査スクリプト
- Before/After 実例

### Section 4: 結論
- **Slide 26-30**: 「パワポを開く前に、Claude Codeを開こう」
